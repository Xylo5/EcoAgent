Eco Project - Context and Updates Summary

================================================================================
CONTEXT
================================================================================
- Game: Eco, a 3D city builder (Clash of Clans style) in Unity (URP + new Input System).
- Core Mechanics: 100x100 grid building placement (place, relocate, demolish), RTS camera.
- Scenes: MainMenu, LevelSelect, Level_0 (Gameplay), MapGenerator (WFC/Grid Generator).
- Assets: SimplePoly City low-poly pack.

================================================================================
PROJECT STRUCTURE
================================================================================

SCENES (Assets/Scenes/):
  - MainMenu.unity        — Title screen with Start and Quit buttons.
  - LevelSelect.unity     — Level selection with 5 buttons (only Level 1 enabled).
  - Level_0.unity          — Main gameplay scene with grid, camera, building system.
  - MapGenerator.unity     — WFC procedural city generation scene.

SCRIPTABLE OBJECT ASSETS:
  Buildings (Assets/):
    - Hut.asset, House.asset, Office.asset, Fact.asset (BuildingData)
  City Tiles (Assets/CityData/):
    - Road.asset, Residential.asset, Industrial.asset, Park.asset, EmptyLot.asset (CityTileData)
    - DefaultAdjacencyRules.asset (CityTileAdjacencyRule)
  Environment (Assets/EnvironmentData/):
    - Mountain.asset, Pond.asset, Tree.asset, Road.asset, RiverSegment.asset (EnvironmentData)

PREFABS (Assets/Prefabs/):
  Buildings: PH1-Hut, PH2-House, PH3-Office, PH4-Fact, ShopButton
  Environment: PH-Mountain, PH-Pond, PH-Tree, PH-Road, PH-RiverSegment

RECOVERY:
  - Assets/_Recovery/0.unity — Scene backup.

================================================================================
SCRIPTS (Assets/Scripts/) — 20 Scripts
================================================================================

1. BuildingData.cs (ScriptableObject)
   - Data container for building types.
   - Fields: buildingName, icon, sizeInCells (1–15, square), prefab, validColor, invalidColor,
     pollutionValue (positive = pollutes, negative = absorbs).

2. BuildingPlacer.cs (MonoBehaviour)
   - Core building placement system with 3 states: Idle, PlacingNew, MovingExisting.
   - Dual input: arrow keys (0.3s initial delay, 0.1s repeat) + mouse terrain raycasting.
   - Ghost building lifted 0.5 units; 2-frame Enter cooldown prevents instant confirm.
   - Disables colliders on ghost during placement.

3. BuildingUI.cs (MonoBehaviour)
   - Two-level shop: fixed "Buildings" button (top-right, green, 140x45) + submenu panel.
   - Button text shows "Name\n(SIZExSIZE)"; selected button scales to 1.15x.
   - Controls: Tab/Shift+Tab to cycle, Enter to select, Escape to close submenu.
   - Submenu auto-closes after placement.

4. CameraController.cs (MonoBehaviour)
   - RTS camera: WASD + middle-mouse pan (speed 40), Z/X + scroll zoom (range 5–300, speed 40),
     Q/E + right-mouse rotate (speed 80). Start pitch 55°.
   - Auto-finds terrain; pan limits = terrain bounds ± 20% margin.

5. CityTileAdjacencyRule.cs (ScriptableObject)
   - Defines WFC adjacency constraints. Maps each CityTileData to allowed cardinal neighbors.

6. CityTileData.cs (ScriptableObject)
   - City tile definition with sizeInCells (1–10) and prefabs[] array for random variants.
   - GetRandomPrefab() picks a random variant.

7. EnvironmentData.cs (ScriptableObject)
   - Permanent environment object definition: objectName, sizeInCells (1–5), prefab.
   - Permanently blocks grid cells; cannot be moved or demolished.

8. EnvironmentObject.cs (MonoBehaviour)
   - Runtime component tracking placed environment objects (gridCell, sizeInCells, environmentData).

9. EnvironmentSpawner.cs (MonoBehaviour)
   - Spawns configured EnvironmentPlacement list at scene start.
   - Marks cells as permanent; validates area before placing. Names objects "Env_<name>".

10. GridCellHighlighter.cs (MonoBehaviour)
    - Visual feedback: green quads (available, 0/0.8/0/0.45) and red quads (occupied, 0.9/0/0/0.45).
    - Object pool of max 25 quads; MaterialPropertyBlock for per-frame color. Quad Y offset 0.08.
    - Respects ignoredBuilding for relocation.

11. GridManager.cs (MonoBehaviour)
    - Core 100x100 grid, fixed cellSize = 2.5 (forced in Awake).
    - Dual tracking: occupiedCells (temporary) + permanentCells (immovable).
    - Grid lines rendered with LineRenderer (green 0.2/1/0.3/0.5, width 0.06, Y offset 0.05).
    - Key methods: IsCellAvailable, IsAreaAvailable (with ignore for relocation), OccupyCells,
      FreeCells, OccupyCellsPermanent, WorldToGrid, GetCellWorldCenter, SnapToGrid.

12. InputManager.cs (MonoBehaviour, Singleton)
    - Centralized input using new Input System with per-frame device caching.
    - Pan: WASD + middle-mouse (invertMousePan = true).
    - Zoom: Z/X + scroll (mouseZoomSensitivity = 10).
    - Rotate: Q/E + right-mouse (mouseRotateSensitivity = 0.5).
    - Actions: Enter (keyboard + left-click), Escape, Tab/Shift+Tab, arrow keys (held + pressed).

13. LevelHUD.cs (MonoBehaviour)
    - In-game HUD with Back button. Escape key returns to level select.

14. LevelSelectUI.cs (MonoBehaviour)
    - Level selection screen with 5 buttons (only Level 1 enabled).
    - Arrow keys/Tab to navigate, Enter to select, Escape to go back.
    - Highlight colors for enabled/disabled/selected states.

15. MainMenuUI.cs (MonoBehaviour)
    - Main menu: Enter to start, Escape to quit.

16. PlacedBuilding.cs (MonoBehaviour, requires BoxCollider)
    - Runtime component for placed buildings with relocation support.
    - PickUp(): frees cells, lifts 1.5 units, stores originalPosition/originalGridCell.
    - PutDown(): re-occupies cells at new position.
    - CancelMove(): returns to original position. Demolish(): frees cells + destroys.

17. PlacementConfirmUI.cs (MonoBehaviour)
    - Keyboard hints panel: "Arrow Keys: Move | Enter: Place | Esc: Cancel".
    - Status: "✓ Valid Position" (green) or "✗ Blocked!" (red) with HTML color tags.
    - Panel follows ghost building, 70px below screen position.

18. SceneLoader.cs (static utility)
    - Scene navigation: LoadMainMenu(), LoadLevelSelect(), LoadLevel(int).

19. WFCCityGenerator.cs (MonoBehaviour)
    - Randomized grid-based city generator (not true WFC — random placement with constraints).
    - Reads each tile's sizeInCells; checks grid bounds and overlap before placing.
    - maxAttempts = gridWidth * gridHeight * maxRetries (default 10).
    - Seed support (0 = random). Generated tiles placed on "Ignore Raycast" layer.
    - Marks cells permanent at runtime via GridManager.OccupyCellsPermanent().

20. PollutionManager.cs (MonoBehaviour, Singleton)
    - Tracks total pollution starting at 100 (configurable).
    - AddPollution(int) updates counter; called by BuildingPlacer on placement.
    - UI text auto-anchored to top-center, color-coded: green (<0), yellow (0–49), red (>=50).

================================================================================
EDITOR SCRIPTS (Assets/Editor/) — 8 Scripts
================================================================================

1. AddCityGeneratorToLevel.cs (EditorWindow)
   - Menu: Tools > EcoAgent > Setup City Generator in Level_0.
   - Opens Level_0, creates/finds WFCCityGenerator GO, wires GridManager + terrain +
     DefaultAdjacencyRules, saves scene.

2. AssignSimplePolyTextures.cs (EditorWindow)
   - Menu: Tools > SimplePoly > Assign All Textures.
   - Auto-matches SimplePoly material names to PNG textures; assigns _BaseMap and _MainTex.
   - Handles naming variants (e.g. _color01 → _1).

3. CityPlaceholderCreator.cs (EditorWindow)
   - Menu: Tools > EcoAgent > Create City Tile Assets.
   - Creates 5 CityTileData assets (Road, Residential, Industrial, Park, EmptyLot) and
     DefaultAdjacencyRules with predefined adjacency rules for WFC generation.

4. EnvironmentPlaceholderCreator.cs (EditorWindow)
   - Menu: Tools > EcoAgent > Create Environment Placeholders.
   - Generates 5 environment prefabs (Mountain, Tree, RiverSegment, Pond, Road) from Unity
     primitives with colors, plus matching EnvironmentData ScriptableObjects.

5. PrefabSizeMeasurer.cs (static utility)
   - Menu: Tools > Measure SimplePoly Prefabs.
   - Instantiates each SimplePoly prefab, calculates renderer bounds, logs dimensions and
     estimated cell counts (based on 2.5 cell size) to console.

6. SceneBuilderEditor.cs (EditorWindow)
   - Menu: Tools > Build All Scenes.
   - Full scene setup: creates MainMenu (title, start/quit buttons) and LevelSelect (5 level
     buttons, grid layout, back button) scenes with TextMeshPro UI; adds LevelHUD to Level_0;
     registers all scenes in Build Settings. Green/blue theme.

7. SimplePolySetup.cs (EditorWindow)
   - Menu: Tools > Apply SimplePoly Asset Prefabs.
   - Batch-updates BuildingData (Hut, House, Office, Factory), EnvironmentData (Mountain, Pond,
     Tree, Road, RiverSegment), and CityTileData assets with SimplePoly prefabs.
   - Auto-calculates sizeInCells from real-world 3D bounds (CellSize = 2.5).

8. WFCCityGeneratorEditor.cs (CustomEditor)
   - Custom Inspector for WFCCityGenerator.
   - Shows GridManager info (dimensions + cell size), "Generate City" (green) and
     "Clear City" (red) buttons. Registers undo/redo.

     without going through MainMenu. Also initializes Keyboard/Mouse device refs in Awake()

   - BuildingPlacer.cs: Added EventSystem.IsPointerOverGameObject() guard in HandleConfirmCancel()

   - MainMenuUI.cs: Changed Update() to use GetEnterKeyDown() instead of GetEnterDown().
